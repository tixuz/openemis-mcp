# Просмотр расписания для класса или учителя

> 📖 **Read-only server.** Playbooks that create or update records require **[openemis-mcp-pro](https://github.com/tixuz/openemis-mcp-pro)**.


**Домен:** Расписание · **Аудитория:** учитель, администратор, студент

Получите еженедельное расписание для конкретного классного звена учреждения — включая день, занятийный слот, предмет и назначенного учителя — для текущего или указанного академического периода. Расписание хранится в `institution-schedule-timetables`; сопоставление класса и оценки из `institution-class-grades` используется для обогащения ответа контекстом образовательной оценки. Для личного просмотра расписания студента (для родителя или самого студента) используйте плейбук `view-student-timetable`.

---

## Ресурсы

| Ресурс | Роль в этом рабочем процессе |
|---|---|
| `institution-schedule-timetables` | Само расписание — каждая строка представляет запланированный слот с днем недели, номером периода, id предмета и назначением учителя. Также используется на шаге 3 для проверки незапланированных пропусков. |
| `institution-class-grades` | Предоставляет образовательную оценку, связанную с этим классом, используется для маркировки отображения расписания контекстом оценки. |

---

## Шаги

| # | Название | Инструмент | Назначение |
|---|---|---|---|
| 1 | Получить расписание класса | `openemis_get` | Получите все запланированные слоты для указанного класса и академического периода. |
| 2 | Получить информацию об оценке для класса | `openemis_get` | Получите образовательную оценку, связанную с этим классом, чтобы обогатить отображение расписания. |
| 3 | Определить пробелы или незапланированные периоды | `openemis_discover` | Проверьте наличие пропущенных занятий, сравнивая полученные строки с ожидаемыми периодами в день. |

### Примечания к шагам

**Шаг 1 — Получить расписание класса:** Отфильтруйте `institution-schedule-timetables` по `institution_class_id` и `academic_period_id`. Если пользователь предоставляет имя учителя вместо класса, сначала разрешите `staff_id` из ресурса персонала, а затем отфильтруйте расписания по `staff_id`. Каждая строка включает `day_of_week` (целое число от 1 до 7), `period` (целое число), `institution_subject_id` и `staff_id`. Упорядочите по `day_of_week` и `period`, чтобы получить читаемую еженедельную сетку.

**Шаг 2 — Получить информацию об оценке для класса:** Отфильтруйте `institution-class-grades` по `institution_class_id`. Зафиксируйте `education_grade_id`, чтобы пометить вывод расписания (например, «Расписание 4 класса — Класс А»). В конфигурациях многоклассных классов может быть возвращено несколько строк — отобразите все связанные оценки.

**Шаг 3 — Определить пробелы или незапланированные периоды:** Сгруппируйте результаты шага 1 по `day_of_week` и перечислите присутствующие номера периодов для каждого дня. Если отсутствуют последовательные периоды (например, во вторник есть периоды 1, 2, 4, но не 3), отметьте их как незапланированные или свободные периоды в выводе. Это рассчитывается вызывающей стороной — в самом ресурсе нет поля «пробел».

## Пример запроса

> «Как выглядит еженедельное расписание класса 9А на этот учебный год?»

Агент выполнит следующее:
1. Вызовет `openemis_discover` или `openemis_list_playbooks`, чтобы найти этот плейбук
2. Выполнит шаги, описанные выше
3. Вернет ответ понятным языком
