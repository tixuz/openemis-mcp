# Генерация отчета о посещаемости ученика (PDF)

**Домен:** Отчет · **Аудитория:** учитель, администратор

Запускает асинхронный генератор отчетов о посещаемости для одного ученика. Возвращает идентификатор процесса, который вызывающая сторона опрашивает до тех пор, пока PDF не будет готов к загрузке. Это двухфазный рабочий процесс: запуск генерации, затем ожидание статуса "завершено" перед получением ссылки на загрузку.

---

## Ресурсы

| Ресурс | Роль в этом рабочем процессе |
|---|---|
| `institution-students-report-cards` | Хранит финальную запись отчета о посещаемости после завершения генерации, включая URL для загрузки PDF. |
| `institution-report-cards` | Перечисляет шаблоны отчетов о посещаемости, настроенные для учреждения и академического периода — отправная точка для определения шаблона, который нужно сгенерировать. |
| `institution-report-card-processes` | Целевой объект для записи, и опрашиваемый целевой объект для проверки статуса — каждая строка отслеживает одну задачу генерации. |
| `report-card-processes` | Системные записи процессов для задач генерации отчетов о посещаемости, используются как альтернативный или дополнительный конечный пункт для проверки статуса. |
| `report-card-email-processes` | Отслеживает задачи доставки электронной почты, если отчет о посещаемости настроен на отправку опекунам после генерации. |
| `report-card-subjects` | Перечисляет, какие предметы включены в шаблон отчета о посещаемости — полезный контекст перед запуском генерации. |
| `report-card-comment-codes` | Справочник кодов комментариев, которые могут появиться в сгенерированном отчете о посещаемости (например, повествовательные коды учителя). |
| `institution-students-report-cards-comments` | Хранит комментарии учителя, прикрепленные к отчету о посещаемости этого ученика и включенные в выходной PDF. |

## Шаги

| # | Название | Инструмент | Цель |
|---|---|---|---|
| 1 | Определить настроенный отчет о посещаемости | `openemis_get` | Найти шаблон отчета о посещаемости для этого учреждения и академического периода, чтобы получить `report_card_id`. |
| 2 | Запустить генерацию | `openemis_create` | POST в `institution-report-card-processes`, чтобы начать асинхронную задачу генерации PDF. |
| 3 | Опрашивать о завершении | `openemis_get` | Периодически проверять статус процесса, пока он не достигнет состояния "завершено". |
| 4 | Получить URL PDF | `openemis_get` | После завершения получить ссылку для загрузки из `institution-students-report-cards`. |

### Примечания к шагам

**Шаг 1 — Определить настроенный отчет о посещаемости:** Вызвать `institution-report-cards`, отфильтрованный по `institution_id` и `academic_period_id`. Если для одного периода существует несколько шаблонов (например, по одному на класс), сузить поиск по `education_grade_id` или спросить пользователя, какой использовать. Запомнить `report_card_id` — он необходим как для вызова генерации, так и для финального получения.

**Шаг 2 — Запустить генерацию:** POST в `institution-report-card-processes` с телами запроса, содержащими `institution_id`, `student_id` и `report_card_id`. В ответ будет включен идентификатор процесса — сохранить его для этапа опроса.

**Шаг 3 — Опрашивать о завершении:** Вызвать `institution-report-card-processes` с идентификатором процесса, чтобы проверить поле статуса. Генерация является асинхронной — опрашивать через разумный интервал (например, каждые несколько секунд), пока статус не станет "completed" или не будет возвращено ошибочное состояние.

**Шаг 4 — Получить URL PDF:** Вызвать `institution-students-report-cards`, отфильтрованный по `student_id` и `report_card_id`. Завершенная запись будет содержать URL или путь к файлу для загрузки сгенерированного PDF.

## Пример запроса

> "Сгенерировать PDF отчета о посещаемости для ученика Мариам за текущий семестр."

Агент будет:
1. Вызвать `openemis_discover` или `openemis_list_playbooks`, чтобы найти этот сценарий
2. Следовать вышеуказанным шагам
3. Вернуть ответ на простом английском языке
