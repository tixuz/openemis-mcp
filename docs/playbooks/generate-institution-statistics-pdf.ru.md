# Сгенерировать отчет в формате PDF со статистикой учреждения

> 📖 **Read-only server.** Playbooks that create or update records require **[openemis-mcp-pro](https://github.com/tixuz/openemis-mcp-pro)**.

**Домен:** Отчет · **Аудитория:** администратор

Администратор запрашивает сводный PDF-отчет по одному учреждению, включающий статистику посещаемости, персонала и присутствия. Этот сценарий описывает шаги по поиску данных; фактический PDF генерируется внутри приложения OpenEMIS.

---

## Ресурсы

| Ресурс | Роль в этом рабочем процессе |
|---|---|
| `institutions` | Предоставляет идентификатор и профиль учреждения — основу для всех запросов статистики. |
| `report-card-processes` | Системные записи процессов для заданий по генерации отчетов — используются для отслеживания статуса задания по генерации статистики, если оно инициировано. |
| `institution-report-card-processes` | Записи процессов, ограниченные учреждением, которые отслеживают статус генерации PDF для этого конкретного учреждения. |

---

## Шаги

| # | Название | Инструмент | Назначение |
|---|---|---|---|
| 1 | Определить шаблон отчета | `openemis_get` | Поиск пользовательского шаблона отчета, настроенного для статистики учреждения. |
| 2 | Начать генерацию | `openemis_create` | POST в `report-card-processes` с идентификатором шаблона и идентификатором учреждения для запуска генерации PDF. |
| 3 | Опрос и загрузка | `openemis_get` | Периодическая проверка статуса процесса до тех пор, пока статус не достигнет «завершено», затем получение ссылки на загрузку. |

### Примечания к шагам

**Шаг 1 — Определить шаблон отчета:** Обратите внимание, что `custom-reports` недоступен через v5 REST API. Генерация PDF статистики использует собственный движок отчетов приложения OpenEMIS. Этот шаг должен включать проверку `report-card-processes` или `institution-report-card-processes` на наличие доступных шаблонов, настроенных для этого учреждения.

**Шаг 2 — Начать генерацию:** Если шаблон доступен через v5 API, выполните POST в `report-card-processes` с идентификатором шаблона, идентификатором учреждения и любыми параметрами диапазона дат. Если шаблон недоступен через v5, направьте пользователя к инициированию PDF из пользовательского интерфейса приложения.

**Шаг 3 — Опрос и загрузка:** Как только получен идентификатор процесса, вызовите `institution-report-card-processes` с этим идентификатором для проверки его статуса. Когда статус достигнет «завершено», в ответе будет включен URL или путь к файлу сгенерированного PDF.

## Пример запроса

> «Сгенерируйте годовой отчет в формате PDF по статистике начальной школы Линкольна».

Агент выполнит следующее:
1. Вызовет `openemis_discover` или `openemis_list_playbooks`, чтобы найти этот сценарий
2. Выполнит шаги, описанные выше
3. Вернет ответ понятным языком

## Примечания

ПРИМЕЧАНИЕ: `custom-reports` недоступен через v5 REST API. Генерация PDF статистики использует движок отчетов внутри приложения. Этот сценарий охватывает только поиск данных — инициируйте PDF из пользовательского интерфейса приложения.
