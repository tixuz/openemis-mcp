# Поиск отметок и результатов оценивания студента по предмету или периоду

> 📖 **Read-only server.** Playbooks that create or update records require **[openemis-mcp-pro](https://github.com/tixuz/openemis-mcp-pro)**.

**Домен:** Оценивание · **Аудитория:** учитель, администратор, родитель

Получите числовые результаты оценивания студента из `assessment-item-results` для одной или нескольких оценок в заданный академический период. Этот плейбук сначала находит соответствующие определения оценивания и идентификаторы периода, затем извлекает необработанные строки отметок и, наконец, обогащает их названиями предметов из `education-subjects`. Результаты включают числовую отметку плюс `grading_option_id`, который можно сопоставить с таблицей оценивания оценки для получения буквенной отметки или статуса «зачтено/не зачтено».

---

## Ресурсы

| Ресурс | Роль в этом рабочем процессе |
|---|---|
| `institution-students` | Подтверждает зачисление студента и предоставляет `institution_id`, `education_grade_id` и `academic_period_id`, необходимые для правильной фильтрации оценок. |
| `assessments` | Перечисляет доступные оценки (промежуточный контроль, итоговый экзамен, четверть) для класса и академического периода студента. |
| `assessment-periods` | Предоставляет метки периодов (Семестр 1, Семестр 2) в рамках каждой оценки, чтобы отметки можно было отобразить с правильным названием периода. |
| `assessment-items` | Перечисляет индивидуальные компоненты предмета в рамках оценки, включая максимальную отметку и `education_subject_id` для разрешения названия предмета. |
| `assessment-item-results` | Источник фактических числовых отметок — одна строка на студента для каждого компонента оценки. |
| `education-subjects` | Справочная таблица названий предметов — используется для замены значений `education_subject_id` на читаемые метки в конечном выводе. |

## Шаги

| # | Название | Инструмент | Цель |
|---|---|---|---|
| 1 | Подтверждение зачисления студента | `openemis_get` | Проверить зачисление студента и получить идентификаторы класса и периода, необходимые для фильтрации оценок. |
| 2 | Получение оценок по классу и периоду | `openemis_get` | Перечислить оценки, доступные для класса и академического периода этого студента. |
| 3 | Получение периодов оценки | `openemis_discover` | Разрешить метки периода оценки, чтобы отметки можно было отобразить с правильным названием периода. |
| 4 | Получение компонентов оценки (Предметов) | `openemis_get` | Получить оцениваемые предметы в рамках выбранной оценки, включая максимальные отметки. |
| 5 | Разрешение названий предметов | `openemis_discover` | Сопоставить значения `education_subject_id` с читаемыми названиями предметов. |
| 6 | Получение отметок студента | `openemis_get` | Получить фактические числовые отметки этого студента по всем компонентам оценки. |

### Примечания к шагам

**Шаг 1 — Подтверждение зачисления студента:** Фильтруйте `institution-students` по `student_id` и `student_status_id=1`. Если пользователь предоставил имя студента вместо идентификатора, выполните поиск по фильтру `first_name` или `last_name`. Зафиксируйте `education_grade_id` и `academic_period_id` из ответа — это ключи соединения для оценок.

**Шаг 2 — Получение оценок по классу и периоду:** Фильтруйте `assessments` по `education_grade_id` и `academic_period_id`, полученным на шаге 1. Если пользователь указал конкретную оценку (например, «итоговый экзамен»), сопоставьте по названию. Если не указано, верните все и позвольте пользователю выбрать или получите результаты для всех.

**Шаг 3 — Получение периодов оценки:** Фильтруйте `assessment-periods` по `assessment_id`, полученному на шаге 2, если он доступен. Сопоставьте `assessment_period_id` каждого результата с этими записями, чтобы отобразить читаемую метку периода рядом с отметками.

**Шаг 4 — Получение компонентов оценки (Предметов):** Фильтруйте `assessment-items` по `assessment_id`. Каждый компонент имеет `education_subject_id` — зафиксируйте их для разрешения названия предмета на шаге 5. В ответе также включен порог проходного балла, если он настроен.

**Шаг 5 — Разрешение названий предметов:** Получите `education-subjects` без фильтров или отфильтруйте по `education_grade_id`, чтобы ограничить набор результатов. Создайте справочную карту вида `{id → name}`, чтобы обогатить строки отметок на шаге 6.

**Шаг 6 — Получение отметок студента:** Фильтруйте `assessment-item-results` по `student_id` и `assessment_id`. Опционально фильтруйте по `assessment_period_id`, если пользователь запросил конкретный семестр. Каждая строка содержит `assessment_item_id` (соединение с шагом 4 для предмета), `marks` (числовая оценка) и `grading_option_id` (ссылка на буквенную оценку). Отсутствие строки для предмета означает, что отметка еще не введена — отличать это нужно от нуля.

## Пример запроса

> «Какие оценки получил Хиндол на промежуточных экзаменах в этом семестре?»

Агент должен:
1. Вызвать `openemis_discover` или `openemis_list_playbooks`, чтобы найти этот плейбук
2. Следовать приведенным выше шагам
3. Вернуть ответ на простом английском языке
