# Запись о поведении студента

> 📖 **Read-only server.** Playbooks that create or update records require **[openemis-mcp-pro](https://github.com/tixuz/openemis-mcp-pro)**.

**Домен:** Студент · **Аудитория:** учитель, администратор

Зарегистрируйте инцидент поведения студента: найдите категории и классификации серьезности, затем составьте запись об инциденте с описанием и действиями по последующему контролю. Доступно в инструментах v0.3.0 для записи.

---

## Ресурсы

| Ресурс | Роль в этом рабочем процессе |
|---|---|
| `student-behaviour-categories` | Справочный список типов инцидентов (например, Буллинг, Опоздание, Вандализм) — предоставляет `category_id` для прикрепления к записи об инциденте. |
| `student-behaviour-classifications` | Справочный список уровней серьезности (Незначительный, Основной, Критический) — предоставляет `classification_id`, указывающий на серьезность инцидента. |
| `student-behaviours` | Целевой объект для записи — запись об инциденте, связанная со студентом, учреждением и академическим периодом, включая описание и любые последующие действия. |

---

## Шаги

| # | Название | Инструмент | Цель |
|---|---|---|---|
| 1 | Список категорий поведения | `openemis_get` | Получить id категории, соответствующей инциденту (например, Буллинг, Опоздание). |
| 2 | Список классификаций серьезности | `openemis_get` | Получить id уровня серьезности (Незначительный, Основной, Критический). |
| 3 | Запись об инциденте | `openemis_create` | Создать запись о поведении, связанную со студентом, учреждением и академическим периодом. |

### Примечания к шагам

**Шаг 1 — Список категорий поведения:** Получите все строки из `student-behaviour-categories`. Просмотрите названия и отметьте `category_id`, который лучше всего соответствует типу инцидента, регистрируемого в данный момент. Этот id требуется на шаге 3.

**Шаг 2 — Список классификаций серьезности:** Получите все строки из `student-behaviour-classifications`. Отметьте `classification_id` для соответствующего уровня серьезности (Незначительный, Основной или Критический). Этот id требуется на шаге 3.

**Шаг 3 — Запись об инциденте:** Отправьте POST в `student-behaviours` со следующими обязательными полями: `student_id`, `institution_id`, `academic_period_id`, `date_of_behaviour` (ГГГГ-ММ-ДД), `description` (повествование о случившемся), `student_behaviour_category_id` (из шага 1) и `student_behaviour_classification_id` (из шага 2). Необязательные поля: `action_taken` (что предпринял учитель или администратор в ответ) и `follow_up_date` (если запланирован последующий контроль).

## Пример запроса

> «Зарегистрировать незначительный инцидент буллинга студента Джеймса 15 апреля — он подшучивал над одноклассником во время перемены».

Агент выполнит следующее:
1. Вызовет `openemis_discover` или `openemis_list_playbooks`, чтобы найти этот сценарий
2. Выполнит шаги, описанные выше
3. Вернет ответ понятным языком
