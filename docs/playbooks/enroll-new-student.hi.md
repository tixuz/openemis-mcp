# किसी संस्थान में एक नए छात्र का नामांकन करें

> 📖 **Read-only server.** Playbooks that create or update records require **[openemis-mcp-pro](https://github.com/tixuz/openemis-mcp-pro)**.

**डोमेन:** छात्र · **दर्शक:** प्रशासक, रजिस्ट्रार

बाईपास मोड केवल: सीधा नामांकन तभी काम करता है जब सिस्टम कॉन्फ़िगरेशन में "नामांकन वर्कफ़्लो को बाईपास करें" सक्षम हो। यदि प्रवेश → नामांकन वर्कफ़्लो सक्रिय है, तो OpenEMIS एप्लिकेशन का उपयोग करें — वर्कफ़्लो के लिए प्रत्येक चरण पर पर्यवेक्षक की स्वीकृति आवश्यक है और इसे केवल API के माध्यम से संचालित नहीं किया जा सकता।

> ⚠️ **वर्कफ़्लो गेट:** यह प्लेबुक केवल तभी काम करती है जब सिस्टम कॉन्फ़िगरेशन में `प्रशासन → सिस्टम कॉन्फ़िगरेशन → छात्र → नामांकन वर्कफ़्लो को बाईपास करें` सक्षम हो। इसके बजाय आधिकारिक एप्लिकेशन के प्रवेश और नामांकन फ़ॉर्म का उपयोग करें।

## संसाधन

| संसाधन | इस वर्कफ़्लो में भूमिका |
|---|---|
| `security-users` | छात्र के लिए आधार उपयोगकर्ता खाता बनाता है — सभी OpenEMIS उपयोगकर्ता (छात्र, कर्मचारी, अभिभावक) इस टेबल को साझा करते हैं। |
| `user-identities` | वैकल्पिक रूप से एक राष्ट्रीय आईडी, पासपोर्ट, या अन्य पहचान दस्तावेज़ को नए छात्र के उपयोगकर्ता रिकॉर्ड से लिंक करता है। |
| `student-guardians` | वैकल्पिक रूप से नामांकन के बाद एक माता-पिता या अभिभावक रिकॉर्ड को छात्र से लिंक करता है। |
| `education-grades` | संदर्भ तालिका जिसका उपयोग नामांकन रिकॉर्ड के लिए लक्षित ग्रेड नाम (जैसे "ग्रेड 5") को उसके id में हल करने के लिए किया जाता है। |
| `academic-periods` | वर्तमान शैक्षणिक अवधि id प्रदान करता है — प्रत्येक नामांकन रिकॉर्ड पर एक आवश्यक फ़ील्ड और पहली जाँच कि API पहुँच योग्य है। |
| `institution-students` | लेखन लक्ष्य — वह नामांकन रिकॉर्ड जो छात्र के उपयोगकर्ता खाते को संस्थान, ग्रेड और शैक्षणिक अवधि से लिंक करता है। |

## चरण

| # | शीर्षक | टूल | उद्देश्य |
|---|---|---|---|
| 1 | पुष्टि करें कि बाईपास मोड सक्रिय है | `openemis_get` | वर्तमान शैक्षणिक अवधि id प्राप्त करें — किसी भी लेखन का प्रयास करने से पहले पुष्टि करता है कि API पहुँच योग्य है। |
| 2 | लक्षित शिक्षा ग्रेड देखें | `openemis_get` | उस ग्रेड के लिए education_grade_id हल करें जिसमें छात्र शामिल हो रहा है। |
| 3 | छात्र उपयोगकर्ता खाता बनाएँ | `openemis_create` | छात्र के व्यक्तिगत विवरणों के साथ आधार उपयोगकर्ता रिकॉर्ड बनाएँ। |
| 4 | पहचान दस्तावेज़ संलग्न करें (वैकल्पिक) | `openemis_create` | एक राष्ट्रीय आईडी या पासपोर्ट नंबर को छात्र के उपयोगकर्ता रिकॉर्ड से लिंक करें। |
| 5 | संस्थान में छात्र का नामांकन करें | `openemis_create` | वह नामांकन रिकॉर्ड बनाएँ जो छात्र को संस्थान, ग्रेड और अवधि से लिंक करता है। |
| 6 | अभिभावक संपर्क लिंक करें (वैकल्पिक) | `openemis_create` | एक अभिभावक या माता-पिता रिकॉर्ड को छात्र के साथ संबद्ध करें। |

### चरण नोट्स

**चरण 1 — पुष्टि करें कि बाईपास मोड सक्रिय है:** किसी भी लेखन के साथ आगे बढ़ने से पहले प्रशासक के साथ सत्यापित करें कि सिस्टम कॉन्फ़िगरेशन में बाईपास नामांकन वर्कफ़्लो सक्षम है। `academic-periods` को `_conditions=current:1` के साथ कॉल करें या तिथि द्वारा सक्रिय अवधि चुनें। `academic_period_id` नोट करें — यह चरण 5 और 6 में आवश्यक है।

**चरण 2 — लक्षित शिक्षा ग्रेड देखें:** `education-grades` को उस ग्रेड के नाम से मिलान करके फ़िल्टर करें जिसमें छात्र शामिल हो रहा है (जैसे "ग्रेड 5" के लिए फ़िल्टर करें)। चरण 5 में उपयोग के लिए `education_grade_id` नोट करें।

**चरण 3 — छात्र उपयोगकर्ता खाता बनाएँ:** `security-users` पर आवश्यक फ़ील्ड्स: `first_name`, `last_name`, `date_of_birth`, और `gender_id` के साथ POST करें। प्रतिक्रिया नए उपयोगकर्ता id को लौटाती है — इसे चरण 5 के लिए `student_id` के रूप में सहेजें।

**चरण 4 — पहचान दस्तावेज़ संलग्न करें (वैकल्पिक):** `user-identities` पर `user_id` (चरण 3 से), `identity_type_id`, और `number` के साथ POST करें। यदि कोई पहचान दस्तावेज़ प्रदान नहीं किया गया है तो इस चरण को छोड़ दें।

**चरण 5 — संस्थान में छात्र का नामांकन करें:** `institution-students` पर `student_id` (चरण 3 से), `institution_id`, `education_grade_id` (चरण 2 से), `academic_period_id` (चरण 1 से), और `student_status_id: 1` (वर्तमान) के साथ POST करें। यदि आपूर्ति नहीं की गई है तो `start_date` डिफ़ॉल्ट रूप से आज की तिथि होगी।

**चरण 6 — अभिभावक संपर्क लिंक करें (वैकल्पिक):** `student-guardians` पर `student_id`, `guardian_id` (अभिभावक का पूर्व-मौजूद उपयोगकर्ता id), और `guardian_relation_id` के साथ POST करें। यदि कोई अभिभावक जानकारी प्रदान नहीं की गई है तो इस चरण को छोड़ दें।
