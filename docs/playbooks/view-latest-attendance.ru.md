# Чтение статистики посещаемости для класса — сегодня / за неделю / хронические прогульщики

> 📖 **Read-only server.** Playbooks that create or update records require **[openemis-mcp-pro](https://github.com/tixuz/openemis-mcp-pro)**.

**Домен:** Посещаемость · **Аудитория:** учитель, администратор, родитель

Быстрый снимок: кто отсутствовал сегодня, итоги за неделю и хронические прогульщики. OpenEMIS НЕ хранит строку для каждого присутствующего студента — присутствие выводится по принципу отсутствия-по-умолчанию после того, как день отмечен. Корректное чтение посещаемости требует трёх уровней: (1) какие дни фактически были отмечены как перекличка; (2) какие дни имели `no_scheduled_class=99` (праздник/отмена — пропустить для статистики); (3) для оставшихся ОТМЕЧЕННЫХ дней извлечь события отсутствия — каждый зачисленный студент без строки в этот день считается ПРИСУТСТВУЮЩИМ.

---

## Ресурсы

| Ресурс | Роль в этом процессе |
|---|---|
| `institution-class-grades` | Подтверждает образовательный уровень(и), назначенный классу — используется для правильного ограничения запросов посещаемости. |
| `institution-class-students` | Предоставляет полный список класса — знаменатель для расчёта, сколько студентов присутствовало или отсутствовало в любой заданный день. |
| `institution-class-attendance-records` | Агрегат на уровне месяца, показывающий, какие дни были отмечены: NOT_MARKED (перекличка не проводилась), PARTIAL_MARKED (отмечены некоторые периоды) или MARKED (все периоды завершены). Также помечает `no_scheduled_class=99` для праздников. |
| `student-attendance-marked-records` | События отсутствия — каждая строка представляет собой запись об опоздании, отсутствии или уважительной причине для конкретного дня для одного студента. Студенты без строки в ОТМЕЧЕННЫЙ день неявно считаются присутствующими. |
| `absence-types` | Справочная таблица для обозначений типов отсутствия: 1 = Уважительная, 2 = Неуважительная, 3 = Опоздание. Используется для различения типов отсутствия при расчёте статистики. |

## Шаги

| # | Название | Инструмент | Цель |
|---|---|---|---|
| 1 | Определить список класса | `openemis_get` | Получить список класса — перечень студентов, зачисленных в настоящее время, которые составляют знаменатель для посещаемости. |
| 2 | Загрузить справочник типов отсутствия | `openemis_get` | Получить справочник типов отсутствия, чтобы события отсутствия можно было классифицировать как отсутствие, опоздание или уважительную причину. |
| 3 | Проверить, какие дни были отмечены | `openemis_get` | Запросить `institution-class-attendance-records`, чтобы найти, какие дни в диапазоне дат фактически были отмечены как проведённая перекличка. |
| 4 | Извлечь события отсутствия за диапазон дат | `openemis_get` | Получить все строки отсутствия для класса за целевой период. |
| 5 | Применить правило чтения | — | Пропустить дни с `no_scheduled_class=99`; для ОТМЕЧЕННЫХ дней считать строки отсутствия как прогулы и вычислить присутствие-по-умолчанию для всех остальных студентов из списка. |

### Примечания к шагам

**Шаг 1 — Определить список класса:** Вызовите `institution-class-students` с фильтрацией по `institution_class_id`, `academic_period_id` и `student_status_id=1`. Количество возвращённых студентов является знаменателем для всех расчётов процента посещаемости.

**Шаг 2 — Загрузить справочник типов отсутствия:** Вызовите `absence-types` без фильтров — это небольшая справочная таблица. Используйте результат для обозначения каждой строки отсутствия как Уважительная (1), Неуважительная (2) или Опоздание (3) в выводе.

**Шаг 3 — Проверить, какие дни были отмечены:** Вызовите `institution-class-attendance-records` с фильтрацией по `institution_class_id`, `academic_period_id`, а также `year` и `month` для целевого периода. Обратите внимание на поле `state`: дни NOT_MARKED нельзя учитывать (перекличка не проводилась); дни PARTIAL_MARKED учитываются только для отмеченных периодов; дни MARKED полностью завершены. Отфильтруйте все строки, где `no_scheduled_class=99` (праздник, форс-мажор, отмена занятий) — они НЕ должны влиять на процент посещаемости любого студента.

**Шаг 4 — Извлечь события отсутствия за диапазон дат:** Вызовите `student-attendance-marked-records` с фильтрацией по `institution_class_id`, `date_from` и `date_to`, с лимитом 500 или более для больших классов. Эти строки представляют собой только события отсутствия.

**Шаг 5 — Применить правило чтения:** Для каждого ОТМЕЧЕННОГО непраздничного дня количество отсутствующих студентов равно количеству строк в `student-attendance-marked-records` за этот день. Количество присутствующих студентов равно (размер списка) минус (количество отсутствующих). Расчёт "хронического прогульщика" должен использовать ТОЛЬКО отмеченные дни — использование полного календарного количества дней приведёт к занижению показателя присутствия для любого дня, когда учитель забыл провести перекличку.

## Примечания

- Расчёт "хронического прогульщика" должен использовать ТОЛЬКО отмеченные дни — использование полного календарного количества дней (например, 30 дней в апреле) приведёт к занижению показателя присутствия для любого дня, когда учитель забыл провести перекличку.
- Маркеры `no_scheduled_class`: 99 = занятий в этот день не было (праздник, форс-мажор, учитель болен, экскурсия); 0 = занятия были запланированы и состоялись. Читатели должны фильтровать по этому полю.
- Для просмотра родителем или опекуном своего собственного ребёнка: отфильтруйте `student-attendance-marked-records` по `student_id` И `institution_class_id`. Они могут видеть события отсутствия своего ребёнка, но не полный список класса, если разрешения этого не позволяют.
