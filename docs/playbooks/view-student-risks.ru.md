# Просмотр профиля рисков и случаев социального обеспечения студента

> 📖 **Read-only server.** Playbooks that create or update records require **[openemis-mcp-pro](https://github.com/tixuz/openemis-mcp-pro)**.

**Домен:** Студент  
**Аудитория:** администратор, консультант, преподаватель  
**ID руководства:** `view-student-risks`

## Описание

Просмотрите рассчитанный уровень риска студента, отдельные критерии риска, которые на него повлияли, а также любые открытые случаи социального обеспечения или защиты для этого студента. У `institution-risks` составной первичный ключ — нет целочисленного `id`. `institution-cases` контролируется рабочим процессом — GET всегда безопасен, но операции записи должны выполняться через приложение OpenEMIS.

---

## Используемые ресурсы

| Ресурс | Назначение |
|---|---|
| `institution-student-risks` | Общий уровень риска студента за академический период |
| `risks` | Основные определения рисков — названия и что измеряет каждый риск |
| `risk-criterias` | Порог и вес для каждого критерия в рамках риска |
| `student-risks-criterias` | Оценки по каждому критерию для записи о риске этого студента |
| `institution-cases` | Открытые случаи социального обеспечения/защиты для этого студента |
| `case-types` | Глобальный справочник: метки типов случаев |
| `case-priorities` | Глобальный справочник: метки приоритетов |

---

## Шаги

| Шаг | Действие | Ресурс | Назначение |
|---|---|---|---|
| 1 | `openemis_get` | `institution-student-risks` | Получите общий уровень риска студента |
| 2 | `openemis_get` | `risks` | Определите названия и определения рисков |
| 3 | `openemis_get` | `student-risks-criterias` | Разбейте оценку по критериям |
| 4 | `openemis_get` | `institution-cases` | Перечислите открытые случаи социального обеспечения для этого студента |

---

## Примечания к шагам

**Шаг 1 — Уровень риска студента**  
Фильтруйте по `institution_id` И `academic_period_id` И `student_id` (`security_user_id` студента). Оба поля `institution_id` и `academic_period_id` обязательны — пропуск любого из них вернёт неограниченные глобальные результаты по всем учреждениям и периодам. Поле `total_risk` — это рассчитанная совокупная оценка. Запомните поле `id` для шага 3.

**Шаг 2 — Определения рисков**  
Фильтруйте по `academic_period_id` — **обязательно, не может быть пустым** (проверяется на уровне API). Таблица `risks` — это основное определение того, что измеряет каждый тип риска для заданного периода. Определите `risk_id` из шага 1 здесь. Получите `risk-criterias`, отфильтрованные по `risk_id`, чтобы получить `risk_value` (диапазон 1–99) и `threshold` для каждого критерия.

**Шаг 3 — Разбивка по критериям**  
Фильтруйте по `institution_student_risk_id` (`id` из шага 1). Это таблица соединения с составным первичным ключом (`institution_student_risk_id` + `risk_criteria_id`) — **нет отдельного целочисленного `id`**. НЕ выполняйте запрос с параметром `id=` без указания таблицы. Поле `value` содержит оценку для каждого критерия. Определите `risk_criteria_id` через `risk-criterias`, чтобы показать метки и пороги.

**Шаг 4 — Случаи социального обеспечения**  
Фильтруйте по `institution_id`. `institution-cases` **контролируется рабочим процессом** — `status_id` ссылается на `workflow_steps`, а не на простое перечисление. GET всегда безопасен.

> ⚠️ Для создания или обновления случаев используйте приложение OpenEMIS — прямые операции записи через API обходят цепочку утверждения и аудита.

`case_number` генерируется автоматически как `{institution_code}-{date}-{id}`. Определите `case_type_id` через `case-types` и `case_priority_id` через `case-priorities` — оба являются глобальными справочниками FieldOption без фильтров `institution_id` или `academic_period_id`.

---

## Ключевые подводные камни

- **`institution-student-risks`** требует оба параметра `institution_id` И `academic_period_id` — оба обязательны.
- **`risks`** требует `academic_period_id` как обязательное поле.
- **`student-risks-criterias` — это таблица соединения с составным первичным ключом** — нет целочисленного `id`. Фильтруйте только по `institution_student_risk_id`.
- **`institution-cases` контролируется рабочим процессом.** GET безопасен. Операции записи должны выполняться через приложение.
- **Имена внешних ключей:** `case_priority_id` (не `priority_id`), `case_type_id` (не `type_id`).

## Пример запроса

> *"Какой уровень риска у Ахмада в этом году и есть ли для него открытые случаи социального обеспечения?"*

1. `openemis_get { resource: "institution-student-risks", params: { institution_id: 6, academic_period_id: 1, student_id: 102 } }` → total_risk: 72, id: 445
2. `openemis_get { resource: "risks", params: { academic_period_id: 1 } }` → "Риск посещаемости", "Академический риск"
3. `openemis_get { resource: "student-risks-criterias", params: { institution_student_risk_id: 445 } }` → критерий пропусков: 85, критерий оценок: 60
4. `openemis_get { resource: "institution-cases", params: { institution_id: 6 } }` → 1 открытый случай, Приоритет: Высокий, Тип: Социальное обеспечение
