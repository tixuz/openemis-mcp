# Зачисление нового студента в учреждение

> 📖 **Read-only server.** Playbooks that create or update records require **[openemis-mcp-pro](https://github.com/tixuz/openemis-mcp-pro)**.

**Домен:** Студент · **Аудитория:** администратор, регистратор

ТОЛЬКО РЕЖИМ ОБХОДА: прямое зачисление работает только при включенной в системной конфигурации опции «обход рабочего процесса зачисления». Если активен рабочий процесс от принятия до зачисления, используйте приложение OpenEMIS — этот рабочий процесс требует одобрения руководителя на каждом этапе и не может быть выполнен только через API.

> ⚠️ **Ворота рабочего процесса:** Этот сценарий работает только при включении `Administration → System Configuration → Student → Bypass Enrolment Workflow` в системной конфигурации. Вместо этого используйте официальные формы Приема и Зачисления.

---

## Ресурсы

| Ресурс | Роль в этом рабочем процессе |
|---|---|
| `security-users` | Создает базовую учетную запись пользователя для студента — все пользователи OpenEMIS (студенты, персонал, родители) используют эту таблицу. |
| `user-identities` | Опционально связывает национальный идентификационный номер, паспорт или другой документ, удостоверяющий личность, с записью нового студента. |
| `student-guardians` | Опционально связывает запись родителя или опекуна со студентом после зачисления. |
| `education-grades` | Справочная таблица, используемая для сопоставления названия класса (например, «5 класс») с его идентификатором для записи о зачислении. |
| `academic-periods` | Предоставляет идентификатор текущего академического периода — обязательное поле в каждой записи о зачислении и первая проверка доступности API. |
| `institution-students` | Целевая таблица для записи — запись о зачислении, которая связывает учетную запись пользователя студента с учреждением, классом и академическим периодом. |

## Шаги

| # | Название | Инструмент | Цель |
|---|---|---|---|
| 1 | Подтвердить активацию режима обхода | `openemis_get` | Получить идентификатор текущего академического периода — подтверждает доступность API перед попыткой каких-либо операций записи. |
| 2 | Найти целевой класс образования | `openemis_get` | Разрешить `education_grade_id` для класса, в который поступает студент. |
| 3 | Создать учетную запись пользователя студента | `openemis_create` | Создать базовую запись пользователя с личными данными студента. |
| 4 | Прикрепить документ, удостоверяющий личность (опционально) | `openemis_create` | Связать национальный идентификационный номер или номер паспорта с учетной записью студента. |
| 5 | Зачислить студента в учреждение | `openemis_create` | Создать запись о зачислении, связывающую студента с учреждением, классом и периодом. |
| 6 | Связать контакт опекуна (опционально) | `openemis_create` | Ассоциировать запись опекуна или родителя со студентом. |

### Примечания к шагам

**Шаг 1 — Подтвердить активацию режима обхода:** Убедитесь у администратора, что в системной конфигурации включен режим обхода рабочего процесса зачисления, прежде чем приступать к операциям записи. Вызовите `academic-periods` с параметром `_conditions=current:1` или выберите активный период по дате. Запишите `academic_period_id` — он требуется в шагах 5 и 6.

**Шаг 2 — Найти целевой класс образования:** Отфильтруйте `education-grades` по названию, соответствующему классу, в который поступает студент (например, отфильтруйте по «5 класс»). Запишите `education_grade_id` для использования в шаге 5.

**Шаг 3 — Создать учетную запись пользователя студента:** Отправьте POST в `security-users` с обязательными полями: `first_name`, `last_name`, `date_of_birth` и `gender_id`. В ответе будет указан новый идентификатор пользователя — сохраните его как `student_id` для шага 5.

**Шаг 4 — Прикрепить документ, удостоверяющий личность (опционально):** Отправьте POST в `user-identities` с `user_id` (из шага 3), `identity_type_id` и `number`. Пропустите этот шаг, если документ, удостоверяющий личность, не предоставлен.

**Шаг 5 — Зачислить студента в учреждение:** Отправьте POST в `institution-students` с `student_id` (из шага 3), `institution_id`, `education_grade_id` (из шага 2), `academic_period_id` (из шага 1) и `student_status_id: 1` (Текущий). `start_date` по умолчанию устанавливается как сегодняшняя дата, если не указана.

**Шаг 6 — Связать контакт опекуна (опционально):** Отправьте POST в `student-guardians` с `student_id`, `guardian_id` (заранее существующий идентификатор пользователя опекуна) и `guardian_relation_id`. Пропустите этот шаг, если информация об опекуне не предоставлена.

## Пример запроса

> «Зачислить нового студента Алину Иванову, родилась 15.03.2014 г., женский пол, в 4 класс в начальную школу Аворы.»

Агент выполнит следующее:
1. Вызовет `openemis_discover` или `openemis_list_playbooks`, чтобы найти этот сценарий
2. Выполнит шаги, описанные выше
3. Вернет ответ на простом английском языке
