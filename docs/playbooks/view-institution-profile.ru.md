# Просмотр полного профиля учреждения

> 📖 **Read-only server.** Playbooks that create or update records require **[openemis-mcp-pro](https://github.com/tixuz/openemis-mcp-pro)**.

**Домен:** Учреждение  
**Аудитория:** администратор, родитель, публичный доступ  
**ID сценария:** `view-institution-profile`

## Описание

Просмотр подробной информации об учреждении: основная запись, активные классы, местоположение и контактные лица. Шаги следуют каноническому порядку — сначала получение основной информации об учреждении, затем классов, затем разрешение местоположения, затем контакты.

---

## Используемые ресурсы

| Ресурс | Назначение |
|---|---|
| `institutions` | Основная запись: название, код, адрес, телефон, locality_id, тип учреждения |
| `institution-grades` | Уровни классов, предлагаемые учреждением в активном академическом периоде |
| `institution-localities` | Глобальная справочная таблица: преобразование locality_id в читаемое название местоположения |
| `institution-contact-persons` | Именные контактные лица с должностью, телефоном, мобильным и email |

---

## Шаги

| Шаг | Действие | Ресурс | Назначение |
|---|---|---|---|
| 1 | `openemis_get` | `institutions` | Получение основной записи об учреждении по id, названию или коду |
| 2 | `openemis_get` | `institution-grades` | Список активных уровней классов для этого учреждения и академического периода |
| 3 | `openemis_get` | `institution-localities` | Преобразование locality_id в читаемое название местоположения |
| 4 | `openemis_get` | `institution-contact-persons` | Список именных контактных лиц для учреждения |

---

## Примечания к шагам

**Шаг 1 — Основная запись об учреждении**  
Фильтруйте по `id`, `name` или `code`. Всегда добавляйте `?is_deleted=0`, чтобы исключить мягко удалённые записи — операция `DELETE` на эндпоинте `institutions` устанавливает `is_deleted=1`; строка остаётся в БД. Целое число `locality_id` из ответа будет разрешено на шаге 3. `GET /api/v5/institutions/{id}` возвращает больше деталей, чем эндпоинт списка; предпочтительнее использовать запрос одной записи, когда у вас есть id.

**Шаг 2 — Активные предлагаемые классы**  
Фильтруйте по `institution_id` (из шага 1) и `academic_period_id`. Без обоих фильтров эндпоинт возвращает глобальную выгрузку. Возвращает пустой массив, если классы не настроены — это допустимо для новых учреждений. Используйте `?_fields=id,education_grade_id,start_date,end_date`, чтобы сократить полезную нагрузку.

**Шаг 3 — Разрешение названия местоположения**  
`institution-localities` — это **ГЛОБАЛЬНЫЙ мастер-справочник** — у него **нет** параметра фильтра `institution_id`. Получите все записи (или закэшируйте один раз за сессию), затем сопоставьте `locality_id` учреждения с правильной строкой. НЕ пытайтесь использовать `?institution_id=...` — этот параметр не поддерживается для этого ресурса и вернёт пустой или неотфильтрованный результат.

**Шаг 4 — Контактные лица**  
Фильтруйте по `institution_id`. Запись с `preferred=1` является основным контактом (`preferred` хранится как tinyint 0/1, приводится к boolean для отображения). Пустой результат допустим — у некоторых учреждений нет отдельных контактов помимо поля `contact_person` в самой записи `institutions`.

## Ключевые подводные камни

- **`is_deleted=0` обязателен** при получении учреждений — мягко удалённые школы остаются в БД и будут появляться в результатах списка без этого фильтра.
- **`institution-localities` НЕ имеет фильтра по конкретному учреждению.** Это самостоятельная справочная таблица. Чтобы найти, к какому местоположению относится учреждение, прочитайте `locality_id` из записи учреждения и найдите его в этой таблице.
- **`contact_person` в записи учреждения — это произвольная текстовая строка**, НЕ внешний ключ на `security_users`. Отдельные, более подробные контактные записи находятся в `institution-contact-persons` с фильтром по `institution_id`.
- **Порядок шагов намеренный:** сначала основная запись (предоставляет `locality_id` и `institution_id` для последующих шагов), затем классы (зависит от `institution_id`), затем местоположение (глобальный поиск, требующий только `locality_id` из шага 1), затем контакты (зависит от `institution_id`).

## Пример запроса

> *"Покажите мне полный профиль для Avory Primary School — предлагаемые классы, местоположение и кому звонить."*

1. `openemis_get { resource: "institutions", params: { name: "Avory Primary", is_deleted: 0 } }` → id=6, locality_id=2
2. `openemis_get { resource: "institution-grades", params: { institution_id: 6, academic_period_id: 1 } }` → классы 1–6
3. `openemis_get { resource: "institution-localities" }` → сопоставить locality_id=2 → "Selangor"
4. `openemis_get { resource: "institution-contact-persons", params: { institution_id: 6 } }` → Jane Doe, Principal
