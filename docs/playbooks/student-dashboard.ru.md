# Панель родителя/ученика — Последнее посещение, поведение, оценки

> 📖 **Read-only server.** Playbooks that create or update records require **[openemis-mcp-pro](https://github.com/tixuz/openemis-mcp-pro)**.

**Домен:** Ученик · **Аудитория:** родитель, ученик

Собрать три самых частых вопроса родителей: как часто мой ребенок отсутствовал в последнее время, были ли инциденты с поведением и какие у него недавние оценки или результаты проверок. Этот сценарий собирает все три набора данных в компактной последовательности и возвращает единый сводный вид.

---

## Ресурсы

| Ресурс | Роль в этом сценарии |
|---|---|
| `institution-students` | Подтверждает текущие данные об обучении ученика — учреждение, класс, группу и академический период — как основу для всех последующих запросов. |
| `student-attendance-marked-records` | Предоставляет информацию о недавних пропусках и опозданиях ученика (отсутствие записи означает присутствие в дни, когда нет пропуска). |
| `student-behaviours` | Перечисляет любые зарегистрированные инциденты с поведением — отсортированы по дате, поэтому наиболее недавние события появляются первыми. |
| `assessment-item-results` | Содержит оценки ученика по каждому предмету и оцениваемому элементу за текущий период. |
| `report-card-comment-codes` | Необязательная справочная информация для разрешения кодов комментариев в отчете, если родитель спрашивает о комментариях учителя. |

## Шаги

| # | Название | Инструмент | Цель |
|---|---|---|---|
| 1 | Разрешение записи ученика | `openemis_get` | Получить текущие данные об обучении ученика для подтверждения личности и получения идентификаторов, необходимых для последующих вызовов. |
| 2 | Получение последних данных о посещаемости | `openemis_get` | Получить наиболее недавние 30 записей об отсутствии/опозданиях ученика. |
| 3 | Получение инцидентов поведения | `openemis_get` | Получить 10 наиболее недавних инцидентов поведения, зарегистрированных в отношении этого ученика. |
| 4 | Получение последних оценок | `openemis_get` | Получить до 50 результатов оцениваемых элементов для ученика за текущий академический период. |

### Примечания к шагам

**Шаг 1 — Разрешение записи ученика:** Вызвать `institution-students` с идентификатором ученика, чтобы подтвердить его зачисление и получить `institution_id`, `institution_class_id`, `education_grade_id` и `academic_period_id`. Эти идентификаторы являются ключами соединения для всех трех наборов данных.

**Шаг 2 — Получение последних данных о посещаемости:** Вызвать `student-attendance-marked-records`, отфильтрованный по `student_id`, упорядоченный по дате в порядке убывания, с ограничением в 30. Каждая строка представляет собой событие пропуска (absence_type_id: 1 = Оправданный, 2 = Неоправданный, 3 = Опоздание). Дни, когда у ученика нет строки, подразумеваются как присутствие, при условии, что день был отмечен.

**Шаг 3 — Получение инцидентов поведения:** Вызвать `institution-student-behaviours`, отфильтрованный по `student_id`, упорядоченный по дате поведения в порядке убывания, с ограничением в 10. Проверить наличие инцидентов — пустой список является положительным результатом для сообщения родителю.

**Шаг 4 — Получение последних оценок:** Вызвать `assessment-item-results`, отфильтрованный по `student_id` и `academic_period_id`, с ограничением в 50. Каждая строка содержит предмет (через assessment_item_id), числовую оценку и `grading_option_id` для разрешения буквенной оценки.
