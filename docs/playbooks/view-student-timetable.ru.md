# Просмотр расписания ученика (Просмотр для родителя / ученика)

> 📖 **Read-only server.** Playbooks that create or update records require **[openemis-mcp-pro](https://github.com/tixuz/openemis-mcp-pro)**.


**Домен:** Расписание · **Аудитория:** родитель, ученик

При наличии идентификатора студента необходимо вернуть расписание класса — предметы, учителей, аудитории и временные слоты для текущего академического периода. Это личный вид, который увидит ученик или родитель и отображает конкретный класс, в котором состоит студент.

---

## Ресурсы

| Ресурс | Роль в этом рабочем процессе |
|---|---|
| `institution-students` | Предоставляет связь между студентом и его записанным классом, извлекая `institution_class_id`, необходимый для поиска расписания. |
| `institution-class-grades` | Подтверждает оценку образования, связанную с классом, используется для обогащения отображения расписания контекстом оценки. |
| `institution-class-subjects` | Перечисляет предметы, преподаваемые в этом классе, используется для сопоставления записей слотов расписания с названиями предметов. |
| `institution-subject-staff` | Связывает предметы с назначенными учителями, позволяя расписанию показывать, кто преподает в каждом классе. |
| `institution-schedule-timetables` | Само расписание — каждая строка представляет собой один запланированный слот с днем, периодом, предметом, учителем и, опционально, аудиторией. |
| `institution-rooms` | Сопоставляет id аудиторий с названиями или кодами аудиторий, чтобы расписание могло отображать физическое местоположение для каждого слота. |

## Шаги

| # | Название | Инструмент | Цель |
|---|---|---|---|
| 1 | Определить класс студента | `openemis_get` | Найти текущую запись о поступлении студента, чтобы получить его `institution_class_id` и `institution_id`. |
| 2 | Прочитать расписание класса | `openemis_get` | Получить все запланированные слоты для этого класса и академического периода. |
| 3 | Обогатить данными о предметах, персонале и аудитории | `openemis_get` | Объединить названия предметов, имена учителей и названия аудиторий для получения читаемого расписания. |

### Примечания к шагам

**Шаг 1 — Определить класс студента:** Вызовите `institution-students`, отфильтрованный по `student_id` и `academic_period_id`, запросив поля `institution_class_id`, `institution_id` и `education_grade_id`. Используйте `student_status_id=1`, чтобы гарантировать возврат только текущей активной записи, а не исторических данных прошлых лет.

**Шаг 2 — Прочитать расписание класса:** Вызовите `institution-timetable` (или `institution-schedule-timetables`), отфильтрованный по `institution_class_id` и `academic_period_id`. Каждая строка содержит день недели (целое число от 1 до 7), номер периода, id предмета и id персонала. Упорядочите по дню недели и периоду для читаемой сетки.

**Шаг 3 — Обогатить данными о предметах, персонале и аудитории:** Получите названия предметов из `institution-class-subjects` и `institution-subjects`, имена учителей из `institution-subject-staff` и названия аудиторий из `institution-rooms`. Используйте это для замены идентификаторов на понятные человеку метки в итоговом расписании.
