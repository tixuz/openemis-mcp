# Просмотр профиля студента, контактов опекунов и истории отсутствий

> 📖 **Read-only server.** Playbooks that create or update records require **[openemis-mcp-pro](https://github.com/tixuz/openemis-mcp-pro)**.

**Домен:** Студент · **Аудитория:** учитель, администратор

Получите полную информацию о зачислении студента, список его опекунов и историю недавних отсутствий в рамках одного рабочего процесса. Шаг с опекунами обогащает каждую контактную запись типом отношений (например, мать, отец, законный опекун), разрешая таблицу поиска отношений опекунов. История отсутствий извлекается из `student-attendance-marked-records` — каждая строка представляет собой отсутствие или опоздание; дни без строки подразумевают присутствие.

---

## Ресурсы

| Ресурс | Роль в этом рабочем процессе |
|---|---|
| `institution-students` | Предоставляет текущие данные о зачислении студента — учреждение, класс, группу и академический период — в качестве основы для остальной части рабочего процесса. |
| `student-guardians` | Перечисляет всех опекунов, связанных с этим студентом, включая контактные данные и идентификатор типа отношений для каждого. |
| `guardian-relations` | Справочная таблица меток типов отношений (Мать, Отец, Законный опекун и т. д.), используемая для преобразования `guardian_relation_id` в понятное человеку отображаемое имя. |
| `student-attendance-marked-records` | Предоставляет информацию о недавних отсутствиях, опозданиях и оправданных пропусках студента, упорядоченную по дате — отсутствие по умолчанию означает присутствие в дни без записи. |

## Шаги

| # | Название | Инструмент | Цель |
|---|---|---|---|
| 1 | Получение записи о зачислении студента | `openemis_get` | Получите текущие данные о зачислении студента, включая учреждение, класс образования, группу и академический период. |
| 2 | Получение контактов опекунов | `openemis_get` | Получите все записи опекунов, связанные с этим студентом, включая контактные данные и идентификатор типа отношений. |
| 3 | Разрешение типов отношений опекунов | `openemis_discover` | Найдите полный список меток типов отношений, чтобы каждый `guardian_relation_id` мог отображаться как понятная человеку метка. |
| 4 | Получение истории отсутствий | `openemis_get` | Получите информацию о недавних пропусках, опозданиях и оправданных отсутствиях студента для быстрого обзора тенденций посещаемости. |

### Примечания к шагам

**Шаг 1 — Получение записи о зачислении студента:** Фильтруйте `institution-students` по `student_id` и, при необходимости, по `academic_period_id`. Включите поля: `id`, `student_id`, `institution_id`, `institution_class_id`, `education_grade_id`, `academic_period_id`, `student_status_id`. Если не установлен `student_status_id=1`, студент может числиться в нескольких периодах — сузьте поиск до текущего. Зафиксируйте `institution_class_id` и `education_grade_id` для шага истории отсутствий.

**Шаг 2 — Получение контактов опекунов:** Фильтруйте `student-guardians` по `student_id`. Ожидайте такие поля, как `guardian_relation_id`, `first_name`, `last_name`, `contact_number` и `email`. У студента может быть от 0 до 4 опекунов; пустой результат допустим, особенно для старшеклассников. `guardian_relation_id` будет разрешен на следующем шаге.

**Шаг 3 — Разрешение типов отношений опекунов:** Это небольшая справочная таблица — получите `guardian-relations` без каких-либо фильтров. Создайте карту поиска вида `{id → name}` и замените `guardian_relation_id` каждого опекуна его меткой. Пропустите этот шаг, если вызывающая сторона нуждается только в контактных номерах и не должна отображать метку отношений.

**Шаг 4 — Получение истории отсутствий:** Фильтруйте `student-attendance-marked-records` по `student_id`, упорядочивайте по дате в порядке убывания и ограничивайте 30 записями для самых недавних событий. Каждая строка содержит `absence_type_id` (1 = Оправданный, 2 = Неоправданный, 3 = Опоздание) и `date`. Помните: хранятся только исключительные записи — студент, у которого нет строки в указанный ОТМЕЧЕННЫЙ день, считается ПРИСУТСТВОВАВШИМ. Чтобы проверить общие показатели за определенный период, также фильтруйте по `date_from` и `date_to`.

## Пример запроса

> «Покажите профиль Мариам, контактную информацию ее опекунов и как часто она отсутствовала в этом месяце».

Агент должен:
1. Вызвать `openemis_discover` или `openemis_list_playbooks`, чтобы найти этот рабочий процесс
2. Выполнить шаги, описанные выше
3. Вернуть ответ на простом английском языке
