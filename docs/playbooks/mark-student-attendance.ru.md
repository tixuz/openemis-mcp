# Отметка присутствия учеников на уроке (Ежедневный список)

> 📖 **Read-only server.** Playbooks that create or update records require **[openemis-mcp-pro](https://github.com/tixuz/openemis-mcp-pro)**.

**Домен:** Посещаемость · **Аудитория:** учитель, администратор

Ежедневный или по периодам список присутствующих. Учитель говорит естественно («Алекс оправдан, Сэм не оправдан, Джордан опоздал, остальные присутствуют»), а агент разрешает классный список, выполняет нечеткое сопоставление имен, сопоставляет фразы с идентификаторами типа отсутствия и публикует только исключения. Каждый ученик, которого нет в строке, считается ПРИСУТСТВУЮЩИМ по умолчанию — это «инвариант отсутствия по умолчанию». Поддерживаются три режима посещаемости: ДЕНЬ (ежедневный список), ПРЕДМЕТ (по предмету) и ДЕНЬ_И_ПРЕДМЕТ (и то, и другое). Сводный отчет за месяц обновляется автоматически после каждой публикации — вам не следует писать в агрегатную таблицу напрямую.

## Ресурсы

| Resource | Role in this workflow |
|---|---|
| `institution-class-grades` | Подтверждает, к какому образовательному классу относится класс, что необходимо для поиска правильного типа отметки о посещаемости. |
| `institution-students` | Предоставляет классный список — список зачисленных учеников, чьи имена агент будет сопоставлять с отчетом учителя. |
| `absence-types` | Справочная таблица, сопоставляющая идентификаторы типа отсутствия с метками: 1 = Оправданный, 2 = Неоправданный, 3 = Опоздание. |
| `student-attendance-marked-records` | Целевая запись для записи — одна строка на ученика, который является исключением, фиксирует его отсутствие, опоздание или оправданный статус за период. |
| `student-attendance-mark-types` | Определяет, какой режим посещаемости настроен для этого класса и периода (DAY, SUBJECT или DAY_AND_SUBJECT). |
| `student-attendance-per-day-periods` | Перечисляет пронумерованные периоды в учебный день для посещаемости в режиме DAY или DAY_AND_SUBJECT. |
| `institution-class-subjects` | Перечисляет предметы, преподаваемые в этом классе — необходимо для режимов SUBJECT или DAY_AND_SUBJECT, чтобы получить идентификаторы предметов. |
| `institution-subjects` | Предоставляет читаемые человеком названия предметов для сопоставления с идентификаторами предметов из institution-class-subjects. |
| `institution-student-absence-details` | Необязательная вторичная запись — фиксирует причину или комментарий об отсутствии, если учитель его предоставляет. |
| `student-absence-reasons` | Каталог справочных структурированных причин отсутствия (Болезнь, Семейная чрезвычайная ситуация, Погода и т. д.) для сопоставления с комментариями в свободной форме. |
| `institution-class-attendance-records` | Сводный отчет за месяц, который автоматически обновляется бэкендом после каждой публикации — не пишите в него напрямую. |

## Шаги

| # | Title | Tool | Purpose |
|---|---|---|---|
| 1 | Подтвердить только конечную точку v5 | — | Все записи должны идти в POST /api/v5/student-attendance-marked-records — никогда не используйте устаревшие пути v4. |
| 2 | Разрешить класс | `openemis_get` | Найти запись класса учреждения, соответствующую классу учителя для текущего академического периода. |
| 3 | Список зачисленных учеников | `openemis_get` | Получить полный список всех учащихся, зачисленных в этот класс. |
| 4 | Обнаружить режим посещаемости | `openemis_get` | Проверить, использует ли класс режим посещаемости DAY, SUBJECT или DAY_AND_SUBJECT. |
| 5 | Разрешить периоды (режим DAY / DAY_AND_SUBJECT) | `openemis_get` | Получить список пронумерованных периодов учебного дня для этого типа посещаемости. |
| 6 | Разрешить предметы (режим SUBJECT / DAY_AND_SUBJECT) | `openemis_get` | Получить идентификаторы и названия предметов для этого класса, чтобы каждая строка посещаемости включала правильный предмет. |
| 7 | Загрузить типы и причины отсутствия | `openemis_get` | Получить обе справочные таблицы параллельно для поддержки сопоставления фраз на естественном языке. |
| 8 | Построить список исключений | — | Определить только тех учеников, кто не присутствовал, на основе инструкций учителя с использованием нечеткого сопоставления имен. |
| 9 | POST строки исключений | `openemis_create` | Записать одну строку отсутствия на каждого ученика-исключение за период в student-attendance-marked-records. |
| 10 | POST подробности об отсутствии (если указана причина) | `openemis_create` | Для каждого ученика, которому учитель дал причину или комментарий, записать строку в institution-student-absence-details. |

### Примечания к шагам

**Шаг 4 — Обнаружение режима посещаемости:** Это должно быть сделано до построения каких-либо строк. Вызовите `student-attendance-mark-types`, отфильтрованный по `education_grade_id` и `academic_period_id`. Код `student_attendance_type_id` сообщает вам режим: 1 = DAY (subject_id должен быть 0, период варьируется), 2 = SUBJECT (период всегда 1, subject_id — это реальный предмет), DAY_AND_SUBJECT (варьируются оба). Не используйте institution-shift-periods — это только расписание смен и не содержит полей посещаемости.

**Шаг 8 — Построение списка исключений:** Нечетко сопоставьте каждое имя, упомянутое учителем, без учета регистра с полями `first_name` и `last_name` в списке. Если имя может соответствовать двум разным ученикам, попросите учителя уточнить перед публикацией. Все те, кого не упомянули, считаются ПРИСУТСТВУЮЩИМИ по умолчанию — не создавайте для них строк.

**Шаг 9 — POST строки исключений:** Отправьте пакет с одной строкой на каждого ученика-исключение за период. Каждая строка должна содержать: `institution_id`, `academic_period_id`, `institution_class_id`, `education_grade_id`, `date`, `period`, `subject_id`, `student_id`, `absence_type_id` и `no_scheduled_class: 0`. Таблица поддерживает upsert — повторная публикация одного и того же составного ключа обновляет существующую строку.

**Шаг 10 — POST подробности об отсутствии:** Используйте те же поля составного ключа, что и в основной строке, плюс `comment` (свободный текст от учителя) и/или `student_absence_reason_id` (сопоставленный из student-absence-reasons). Пример: "оправдан, потому что его мать больна" → comment = 'mother is ill', student_absence_reason_id = 1 (Illness).

## Пример запроса

> «Отметьте всех присутствующими в 7А сегодня, кроме Сэма, который отсутствует неоправданно, и Джордана, который опоздал».

Агент будет:
1. Вызывать `openemis_discover` или `openemis_list_playbooks`, чтобы найти этот сценарий
2. Следовать вышеуказанным шагам
3. Возвращать ответ на простом английском языке

## Примечания

- Фронтенд (Angular) использует устаревший путь `/institutions/{institution_id}/students/{student_id}/absences`. Агенты должны использовать вышеуказанные маршруты v5 — они являются стабильным контрактом.
- НЕ публикуйте в `institution-class-attendance-records` напрямую для отметки — это агрегат на уровне месяца, который обновляется бэкендом автоматически.
через хук `afterSaveCommit`.
- ОТСУТСТВИЕ ПО УПУСТУ: как только день/период отмечен, ученики, у которых нет строки в `student-attendance-marked-records` за этот день/период, считаются ПРИСУТСТВУЮЩИМИ. Вам нужны строки только для тех учеников, кто опоздал, отсутствовал или был оправдан.
- НЕТ_ЗАПЛАНИРОВАННОГО_КЛАССА для праздников, форс-мажоров или отмененных занятий: ОТПРАВЬТЕ одну строку с `no_scheduled_class = 99` и без строк на каждого ученика. Сигнал (Sentinel) 0 означает, что занятие БЫЛО запланировано и состоялось.
- Если учитель говорит "все присутствуют", день остается НЕ_ОТМЕЧЕННЫМ (нет строк). Чтобы отметить, что день был проведен, опубликуйте хотя бы одну строку или подтвердите с учителем, отсутствовал ли кто-нибудь из учеников.
