# Основная информация об учреждении — количество студентов, сотрудников, вакантных должностей

> 📖 **Read-only server.** Playbooks that create or update records require **[openemis-mcp-pro](https://github.com/tixuz/openemis-mcp-pro)**.

**Домен:** Учреждение · **Аудитория:** admin, parent

Классический запрос типа «расскажи мне об этой школе». Один поиск учреждения с последующими тремя запросами на подсчет, использующими метаданные постраничной навигации для получения итогов без загрузки каждой записи.

---

## Ресурсы

| Resource | Role in this workflow |
|---|---|
| `institutions` | Разрешает название или код учреждения в идентификатор записи и предоставляет базовую профильную информацию о школе. |
| `institution-students` | Предоставляет общее количество зачисленных в настоящее время студентов, считывая метаданные постраничной навигации (last_page), а не загружая все записи. |
| `institution-staff-position-profiles` | Используется для подсчета активных сотрудников в учреждении с помощью того же подхода постраничной навигации. |
| `institution-positions` | Предоставляет количество вакантных должностей в учреждении. |

---

## Шаги

| # | Title | Tool | Purpose |
|---|---|---|---|
| 1 | Разрешить учреждение | `openemis_get` | Поиск учреждения по названию для получения его идентификатора и базового профиля. |
| 2 | Подсчитать текущих студентов | `openemis_get` | Получить одну строку студента для чтения значения last_page, которое равно общему числу текущих зачисленных. |
| 3 | Подсчитать активных сотрудников | `openemis_get` | Получить одну строку сотрудника для чтения значения last_page, которое равно общему числу активных сотрудников. |
| 4 | Подсчитать вакантные должности | `openemis_get` | Получить одну строку должности, отфильтрованную по статусу «вакантно», для чтения количества в last_page. |

### Примечания к шагам

**Шаг 1 — Разрешить учреждение:** Вызвать `institutions`, отфильтровав по `name` (например, "Avory Primary"), запросив только поля `id`, `name` и `code`. Запомните `institution_id` — это ключ фильтра для всех последующих вызовов.

**Шаг 2 — Подсчитать текущих студентов:** Вызвать `institution-students` с `institution_id`, `student_status_id=1` и `limit=1`. Поле `last_page` в ответе (или эквивалентное общее количество постраничной навигации) равно общему числу в настоящее время зачисленных студентов. Вам не нужно перебирать все записи.

**Шаг 3 — Подсчитать активных сотрудников:** Вызвать `institution-staff` с `institution_id`, `status_id=1` и `limit=1`. Прочитать `last_page` для получения общего количества активных сотрудников.

**Шаг 4 — Подсчитать вакантные должности:** Вызвать `institution-positions` с `institution_id`, `status='vacant'` и `limit=1`. Прочитать `last_page` для получения общего количества вакантных должностей.

## Пример запроса

> «Дай мне краткий обзор начальной школы Авори — сколько здесь студентов, сотрудников и вакантных должностей?»

Агент будет:
1. Вызвать `openemis_discover` или `openemis_list_playbooks`, чтобы найти этот сценарий
2. Следовать приведенным выше шагам
3. Вернуть ответ на простом английском языке
