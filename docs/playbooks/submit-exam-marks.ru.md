# Ввод оценок по предметному экзамену за период оценки

> 📖 **Read-only server.** Playbooks that create or update records require **[openemis-mcp-pro](https://github.com/tixuz/openemis-mcp-pro)**.

**Домен:** Оценка · **Аудитория:** учитель

Пакетный ввод оценок для каждого ученика по одному оцениваемому элементу, фиксация и последующая проверка расчета взвешенного результата. Этот сценарий находит нужную оценку и список класса, затем публикует одну строку оценки на каждого ученика.

---

## Ресурсы

| Ресурс | Роль в этом рабочем процессе |
|---|---|
| `assessments` | Перечисляет доступные оценки (семестровый контроль, итоговый экзамен, ежеквартальный) для этой успеваемости и периода — отправная точка для определения, по какой оценке вносить баллы. |
| `assessment-periods` | Предоставляет метки периода (Семестр 1, Семестр 2) в рамках оценки — необходимо для ограничения оценок нужным семестром. |
| `assessment-items` | Перечисляет отдельные компоненты предмета в рамках оценки (например, Математика, Английский), включая максимальные баллы — один элемент на каждый предмет. |
| `assessment-item-results` | Целевая запись — одна строка на ученика по оцениваемому элементу фиксирует его числовой балл и оценку. |
| `education-subjects` | Справочная таблица для названий предметов, используется для отображения, какому предмету соответствует каждый оцениваемый элемент. |
| `institution-class-students` | Список класса — предоставляет список идентификаторов учеников для ввода оценок. |

---

## Шаги

| # | Название | Инструмент | Цель |
|---|---|---|---|
| 1 | Получить оценку и период | `openemis_get` | Найти определение оценки для этой успеваемости и академического периода, чтобы получить `assessment_id` и детали периода. |
| 2 | Получить список класса | `openemis_get` | Получить список текущих зачисленных учеников в классе, чтобы можно было подготовить строку оценки для каждого из них. |
| 3 | Подать балл на ученика | `openemis_create` | Отправить одну строку в `assessment-item-results` на каждого ученика с его числовым баллом. |

### Примечания к шагам

**Шаг 1 — Получить оценку и период:** Вызвать `assessments`, отфильтрованные по `academic_period_id` и `education_grade_id`. Если учитель назвал конкретную оценку (например, "итоговый экзамен"), сопоставить по названию. Запомнить `assessment_id` — он необходим для шагов 2 и 3. Также вызвать `assessment-periods`, отфильтрованные по `assessment_id`, чтобы получить правильный `assessment_period_id`.

**Шаг 2 — Получить список класса:** Вызвать `institution-class-students`, отфильтрованные по `institution_class_id`, `academic_period_id` и `student_status_id=1` (только активные). Каждая строка дает `student_id`, необходимый для подачи оценки.

**Шаг 3 — Подать балл на ученика:** POST в `assessment-item-results` с `assessment_id`, `assessment_period_id`, `assessment_item_id` (идентифицирует предмет), `student_id` и `marks` (числовой балл). Строки можно подавать индивидуально или пачкой. После публикации проверить, что результаты сохранены, вызвав `assessment-item-results`, отфильтрованные по `assessment_id` и `student_id`.

## Пример запроса

> «Введите баллы по математике семестрового контроля для класса 8Б — все ученики набрали от 65 до 90 баллов».

Агент будет:
1. Вызывать `openemis_discover` или `openemis_list_playbooks`, чтобы найти этот сценарий
2. Следовать приведенным выше шагам
3. Возвращать ответ понятным языком
